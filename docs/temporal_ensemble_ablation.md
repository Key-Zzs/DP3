# Temporal Ensemble 真机部署消融实验

## 实验目的

本报告分析部署端 temporal ensemble 是否能减小独立采样的 DP3 action chunk
之间的动作不连续，渐变权重是否能消除固定融合权重引入的新接缝，以及两种融合方式
与可选 200 Hz 笛卡尔 servo 平滑层之间的相互影响。

实验在两种控制链下分别比较三种 temporal ensemble 模式：

1. 不使用 temporal ensemble；
2. 在整个对齐重叠区间内固定使用 `0.5` 的新 chunk 权重；
3. 新 chunk 权重按 `0.50 -> 0.75 -> 1.00` 线性递增。

融合只作用于双臂共 12 维笛卡尔位姿动作。两个夹爪命令始终直接使用新预测，
不参与融合。

## Temporal Ensemble 时间对齐关系

策略输出完整的预测 horizon，而部署端同步执行一个 action chunk。本次实验的模型
契约为：

- `n_obs_steps = 2`；
- `n_action_steps = 4`；
- 可执行动作起始索引为 `1`；
- 相邻两次预测之间有 `3` 个未来动作在时间上对齐。

旧预测未执行的 tail 与新预测的可执行 head 指向相同的未来控制时刻。固定权重实现为：

```text
新 chunk 权重 = [0.50, 0.50, 0.50]
旧 chunk 权重 = [0.50, 0.50, 0.50]
```

渐变权重实现为：

```text
新 chunk 权重 = [0.50, 0.75, 1.00]
旧 chunk 权重 = [0.50, 0.25, 0.00]
```

第 4 个可执行动作在旧预测 horizon 中已经没有可对齐的动作，因此始终完全使用新预测。
固定方案会在可执行动作 2 和动作 3 之间把新预测权重从 `0.5` 突然切换为 `1.0`；
渐变方案在动作 2 时已经达到 `1.0`，不会额外引入这次权重跳变。

配置行为如下：

- `temporal_ensemble_coeff: 0.0` 不读取 `action_pred`，严格保留原始 action queue
  路径；
- 系数大于 0 且 `temporal_ensemble_ramp_weights: false` 时恢复固定权重行为；
- 系数大于 0 且 `temporal_ensemble_ramp_weights: true` 时使用线性渐变权重。

## 分析方法

### 动作不连续指标

从主 JSONL 日志中读取：

- `action_vector.model_raw`：未经融合的原始模型输出；
- `action_vector.temporal_ensemble`：重叠融合后的动作；
- `chunk_index`：当前发送动作在 chunk 中的位置。

相邻动作的平移不连续定义为：

```text
||delta_xyz[t] - delta_xyz[t-1]||_2
```

旋转不连续定义为相邻三个旋转向量通道差值的 L2 范数。

目标动作的 `chunk_index == 0` 时，该动作变化被归类为 chunk 边界跳变；目标动作的
`chunk_index == 3` 时，被归类为 chunk 尾部的第 `2 -> 3` 步跳变。除非另有说明，
报告均采用第 95 百分位数 p95。

在同一次运行内部比较 `model_raw` 和 `temporal_ensemble` 具有最强的因果意义，
因为二者来自完全相同的一次模型采样。不同真机运行之间还会受到观测、扩散随机噪声、
任务阶段和实际机械臂运动差异的影响。

### 延迟指标

主 JSONL 日志提供：

- 点云构建耗时；
- DP3 策略预测耗时；
- 生成新 chunk 时的端到端策略延迟；
- 主循环耗时和队列动作年龄。

实测策略频率由 control trace 中相邻 `policy_action.monotonic_ns` 计算，而不是直接采用
配置文件中的目标频率。

### Command 与 TCP 跟踪误差

每个 TCP 反馈样本与时间上不晚于该反馈的最近一条 command 进行因果配对。为了减少
运行时长不同造成的影响，Direct 组三次运行统一比较前 723 个策略动作，Servo 组三次
运行统一比较前 323 个策略动作。该处理可以避免用很短的初始阶段与完整长轨迹直接比较，
但不能让不同真机运行成为完全相同的受控轨迹。

## 实验结果

### 右臂 action 不连续 p95

| 模式 | chunk 边界平移 | 尾部平移 | 整体平移 | chunk 边界旋转 | 整体旋转 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 不融合，Direct | 1.506 mm | 0.369 mm | 0.973 mm | 0.230° | 0.141° |
| 固定权重，Direct | 0.977 mm | 0.865 mm | 0.678 mm | 0.165° | 0.141° |
| **渐变权重，Direct** | **0.712 mm** | **0.521 mm** | **0.556 mm** | **0.165°** | **0.136°** |
| 不融合，Servo | 2.158 mm | 0.467 mm | 1.147 mm | 0.356° | 0.231° |
| 固定权重，Servo | 0.900 mm | 0.911 mm | 0.632 mm | 0.170° | 0.143° |
| **渐变权重，Servo** | **1.129 mm** | **0.339 mm** | **0.655 mm** | **0.197°** | **0.155°** |

Direct 渐变实验相对 Direct 固定权重实验：

- chunk 边界平移下降 27%；
- 尾部第 `2 -> 3` 步平移下降 40%；
- 所有相邻动作的整体平移下降 18%；
- 整体旋转基本不变，略有改善。

Servo 组中，固定权重与渐变权重的整体动作变化接近。渐变实验的边界值更高，但两种
算法对第一个重叠动作使用的权重完全相同，都是 `0.5`，因此两次独立运行之间的边界差异
不能归因于渐变公式本身。

### 同一次采样中 raw 与 ensemble 的变化

| 运行 | 边界平移 raw -> ensemble | 边界旋转 raw -> ensemble | 整体平移 raw -> ensemble | 整体旋转 raw -> ensemble |
| --- | ---: | ---: | ---: | ---: |
| 固定，Direct | 1.716 -> 0.977 mm（-43%） | 0.316 -> 0.165°（-48%） | 1.040 -> 0.678 mm（-35%） | 0.207 -> 0.141°（-32%） |
| 渐变，Direct | 1.660 -> 0.712 mm（-57%） | 0.318 -> 0.165°（-48%） | 0.880 -> 0.556 mm（-37%） | 0.236 -> 0.136°（-42%） |
| 固定，Servo | 1.891 -> 0.900 mm（-52%） | 0.309 -> 0.170°（-45%） | 0.880 -> 0.632 mm（-28%） | 0.200 -> 0.143°（-28%） |
| 渐变，Servo | 2.083 -> 1.129 mm（-46%） | 0.345 -> 0.197°（-43%） | 0.950 -> 0.655 mm（-31%） | 0.189 -> 0.155°（-18%） |

固定权重和渐变权重在各自运行内部都稳定降低了 chunk 边界跳变和整体动作变化。这是
temporal ensemble 本身有效的最可靠证据。

### 固定权重产生的尾部接缝

| 运行 | raw 尾部平移 | ensemble 尾部平移 | raw 尾部旋转 | ensemble 尾部旋转 |
| --- | ---: | ---: | ---: | ---: |
| 固定，Direct | 0.601 mm | 0.865 mm | 0.103° | 0.163° |
| 渐变，Direct | 0.521 mm | 0.521 mm | 0.155° | 0.155° |
| 固定，Servo | 0.347 mm | 0.911 mm | 0.105° | 0.174° |
| 渐变，Servo | 0.339 mm | 0.339 mm | 0.135° | 0.135° |

固定权重使 Direct 运行的尾部平移不连续增加约 44%，使 Servo 运行增加约 163%。
两次渐变运行中，ensemble 尾部变化与 raw 完全相同，因为新预测权重在该位置之前已经
达到 `1.0`。这证明渐变权重从结构上消除了它所针对的固定权重尾部接缝。

渐变权重并不保证重叠区内的每一个相邻变化都减小。它会把旧预测到新预测之间的修正
分散到重叠区间。在渐变 Servo 实验中，索引 1 和 2 的部分变化有所增加，但边界和整体
变化仍低于该次运行的 raw 输出。

### Command 与 TCP 跟踪

| 模式 | 右臂 command 步长 p95 | 右臂 command 速度 p95 | command 到 TCP 平移误差 p95 |
| --- | ---: | ---: | ---: |
| 不融合，Direct | 2.834 mm | 22.76 mm/s | 2.407 mm |
| 固定权重，Direct | 3.761 mm | 27.01 mm/s | 3.149 mm |
| **渐变权重，Direct** | **2.293 mm** | **20.33 mm/s** | **2.208 mm** |
| 不融合，Servo | 0.565 mm | 110.45 mm/s | 5.151 mm |
| 固定权重，Servo | 0.169 mm | 33.53 mm/s | 5.496 mm |
| **渐变权重，Servo** | **0.430 mm** | **84.49 mm/s** | **5.179 mm** |

渐变 Direct 运行在本组中具有最低的 command 激励和平移跟踪误差。

在 servo 平滑层下，固定权重运行产生的 200 Hz command 步长最小，渐变结果位于固定
权重和无融合之间。渐变 Servo 的平移跟踪误差略低于固定权重，但右臂旋转
command 到 TCP 的 p95 误差更高，为 `0.721°`，固定权重为 `0.426°`。由于真机轨迹
并不相同，这些跨运行跟踪误差只能说明部署趋势，不能作为严格的因果结论。

Servo 层实际运行频率约为 197 Hz。`alpha=0.25` 时，每次 target 更新后的第一条
servo command 大约只执行 target 差值的 25%。其一阶时间常数约为 17.6 ms，达到
固定 target 90% 和 95% 分别约需 41 ms 和 53 ms。该层可以抑制 command 阶跃，
但也会引入相位滞后，且不能修正带有系统偏差的策略动作。

### 延迟

| 模式 | 点云构建 p50 | DP3 预测 p50 | 新 chunk 总延迟 p50 | 实测策略频率 |
| --- | ---: | ---: | ---: | ---: |
| 不融合，Direct | 125.8 ms | 39.2 ms | 168.7 ms | 7.78 Hz |
| 固定权重，Direct | 128.8 ms | 39.7 ms | 168.9 ms | 7.47 Hz |
| **渐变权重，Direct** | **122.8 ms** | **38.0 ms** | **161.8 ms** | **7.89 Hz** |
| 不融合，Servo | 147.1 ms | 42.7 ms | 191.0 ms | 6.54 Hz |
| 固定权重，Servo | 138.3 ms | 40.7 ms | 180.0 ms | 7.02 Hz |
| **渐变权重，Servo** | **131.0 ms** | **39.5 ms** | **171.1 ms** | **7.39 Hz** |

日志中没有测出 temporal ensemble 带来的延迟惩罚。渐变运行比固定运行更快，但其点云
构建也恰好快了约 6 至 7 ms，因此延迟差异来自当次感知运行波动，不能说明渐变融合
会加速推理。

尽管固定和渐变实验配置为 30 Hz，实测策略频率仍只有 7.0 至 7.9 Hz。点云构建本身
的中位耗时约为 123 至 138 ms。四步 chunk 的实际执行时间约为 0.54 至 0.61 秒，
而不是 30 Hz 契约对应的 0.133 秒。队列动作年龄 p95 约为 0.4 秒。该感知和执行频率
失配远大于两种 temporal ensemble 算法自身的延迟影响。

## 实验限制

1. 六次实验是相互独立的真机运行，初始观测、扩散噪声、任务阶段和实际运动轨迹并不
   完全相同。
2. 无融合实验使用旧点云配置和 10 Hz 目标周期；由于所有实验都受感知耗时限制，其
   实测频率仍与新实验接近。
3. 只有同一次运行中的 raw 与 ensemble 对比能够隔离融合算法的数学作用。跨运行 TCP
   跟踪比较只能说明部署趋势。
4. 夹爪命令有意排除在 temporal ensemble 之外，本实验不能说明夹爪抖动是否改善。
5. Control trace 记录的是 50 Hz TCP 反馈，不是机械臂内部 1 kHz 控制状态。

## 结论与部署选择

Temporal ensemble 在没有可测推理开销的情况下，稳定减小了右臂 chunk 边界动作
不连续。固定 `0.5` 权重会在重叠结束处引入新的动作接缝；线性渐变通过在最后一个
重叠动作前达到纯新预测，消除了该结构性问题。

推荐部署配置为：

```yaml
inference:
  temporal_ensemble_coeff: 0.5
  temporal_ensemble_ramp_weights: true
```

受控回退配置为：

```yaml
# 恢复原来的固定权重 temporal ensemble。
temporal_ensemble_coeff: 0.5
temporal_ensemble_ramp_weights: false

# 完全关闭 temporal ensemble，恢复原始 action queue 路径。
temporal_ensemble_coeff: 0.0
```

下一项优化应优先针对点云采样和感知延迟。在当前实际约 7 Hz、队列动作已经陈旧数百
毫秒的情况下，不应依靠更强的平滑或更低的初始融合权重掩盖执行瓶颈，否则会继续增加
闭环相位滞后。
